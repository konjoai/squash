Why this matters

Three questions every team shipping AI eventually has to answer.

📚

Was this model trained on copyrighted data?

The answer used to be "we hope not." Squash gives you a signed report that walks back through the model's family tree to the original training corpus.

🛡️

Can I prove this AI hasn't been tampered with?

Cryptographic attestations bind the model bytes to a signature. If anything changed — weights, config, even a single byte — verification fails loudly.

🎯

Did this output come from the model I think it did?

Every attestation contains the exact SHA-256 of the model that produced it. No room for swap-the-weights attacks in production.

Plain-English features

The technical names live in the source code. Here's what they do for you.

🔐 Tamper-proof certificates
Every signed cert proves the model bytes haven't changed since attestation. SLSA Build L3
⏱️ Trusted timestamps
An external authority co-signs when the cert was issued. Even squash can't back-date. RFC 3161
🧪 Identical every time
Same input → same signed bytes, on any computer, in any year. RFC 8785
🌳 Know your model's history
Which family. Which datasets. Which copyright risks. squash.genealogy
⚖️ Licence safety check
Combine licences and deployment scenarios. Get a verdict + recommendations. squash.copyright
🤖 Automated red-team testing
Mathematical properties checked across millions of inputs every night. Hypothesis
